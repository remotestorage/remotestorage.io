Community

Get help or help out

GitHub

All source code is hosted on GitHub, and we use GitHub's issues, pull requests and comments extensively.

What to use it for:

  • Reporting bugs and problems
  • Asking questions about source code
  • Contributing new source code
  • Suggesting new features or behaviour

IRC

The core developers as well as other community members are usually available for a chat in the #remotestorage channel on Freenode. You can open the chat in your browser, if you don't have an IRC client.

What to use it for:

  • Asking general questions about remoteStorage
  • Telling people about your app/project/...
  • Watercooler talk

StackOverflow

If you need help during the development of your remoteStorage-enabled app, you can ask a question on StackOverflow and tag it with "remotestorage". It is our official support channel for app developers.

What to use it for:

  • Asking questions about remoteStorage app development
  • Helping people with remoteStorage app development

Mailing list

We retired all existing mailing lists in favor of dicussing things via GitHub Issues in the remotestorage.io repository. If you're still using an old mailing list, you probably won't get a response.

What to use it for:

  • Discussing the remoteStorage project
  • Announcing your remoteStorage-enabled app for test and feedback

Twitter

Follow @remotestorage_ on Twitter to receive updates and announcements about releases, events, apps, and related news.

Before creating a dedicated account recently, we used to tweet everything from @unhosted, which you should also check out for general updates on fighting hosted data silos!

Meet people in person

remoteStorage folks are travelling a lot, and it's rumored some of them don't even have a home address. So chances are good you can meet people in person, learn about remoteStorage at a conference, or even be part of a remoteStorage hackathon. Be sure to follow us on Twitter to receive event announcements!

The Unhosted Unconference

Every year in September, we throw an uncoference-style event in a small town called Unhost, right next to Prague. During these days we discuss the whole project, learn about new developments, work on issues, make larger decisions, and most importantly: have a lot of fun and learn to know each other. Check out and track the next one on Lanyrd!
